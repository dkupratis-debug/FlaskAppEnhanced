# Learning Tracks

Use these tracks to guide different learners through the repo.

## Track A: Beginner (No GitHub Experience)

### Goal
Understand GitHub basics and complete one safe practice task.

### Steps
1. Read `README.md`.
2. Follow `docs/FIRST_10_MINUTES.md`.
3. Open pinned issue `#9 Start Here`.
4. Complete practice Exercise 1 in `docs/PRACTICE_EXAMPLES.md`.
5. Open `Actions` and confirm checks are green.

### Completion Criteria
- You can explain what `Issues`, `Pull requests`, and `Actions` are.
- You completed one tiny change in your fork.

## Track B: Intermediate (Can Use Git + PRs)

### Goal
Use the repo workflows like a contributor.

### Steps
1. Complete Exercises 4, 5, and 8 in `docs/PRACTICE_EXAMPLES.md`.
2. Review one PR with `docs/PR_REVIEW_CHECKLIST.md`.
3. Compare two tags using `docs/VERSIONING_AND_RELEASES.md`.
4. Inspect `ci.yml`, `release.yml`, and `package.yml`.

### Completion Criteria
- You can open/review PRs confidently.
- You can explain release/package workflow differences.

## Track C: Maintainer (Trainer/Admin)

### Goal
Run this repo as a professional training environment.

### Steps
1. Read:
   - `ROADMAP.md`
   - `docs/PUBLIC_LAUNCH_CHECKLIST.md`
   - `docs/SAFE_SHARING.md`
   - `docs/ANALYTICS_AND_PRIVACY.md`
2. Create one weekly challenge issue using the template.
3. Use `docs/COACHING_REPLY_TEMPLATES.md` to respond consistently.
4. Check `Insights -> Traffic` and open Issues/PRs weekly.

### Completion Criteria
- You can onboard new learners quickly.
- You can track engagement and support questions consistently.

## Recommended Order for Group Training

1. Week 1: Track A
2. Week 2: Track B (first half)
3. Week 3: Track B (second half)
4. Week 4: Track C style review/demo by maintainer
