# Good First Issues Backlog

Use these starter tasks to onboard beginners safely.

## Starter Tasks

1. Fix one typo in `docs/FIRST_10_MINUTES.md` and open a PR.
1. Add one FAQ entry in `docs/FAQ.md` using the existing format.
1. Add one screenshot placeholder note in `docs/images/README.md`.
1. Improve one command explanation in `README.md`.
1. Add one troubleshooting tip in `docs/CI_TROUBLESHOOTING_FLOW.md`.
1. Add one beginner hint to `docs/PRACTICE_EXAMPLES.md`.
1. Add one label definition in `docs/LABELS_GUIDE.md`.
1. Improve one step in `docs/DASHBOARD_TUTORIAL.md`.
1. Add one glossary term in `docs/GITHUB_GUIDE.md`.
1. Validate one broken link candidate and submit a fix PR.

## Maintainer Notes

- Label these with `good first issue` and `learning`.
- Keep each task scoped to 15-30 minutes.
- Include exact file path and expected outcome in issue body.
- Pin one starter issue at a time to avoid overwhelming newcomers.
